# 파트 1. 아이템 시스템

## 1-1. C++ 인터페이스 (`IItemInterface`)

언리얼에서 인터페이스는 **두 개의 클래스**로 만들어진다는 게 핵심 포인트.

```cpp
UINTERFACE(MinimalAPI)
class UItemInterface : public UInterface   // ← 리플렉션용 빈 껍데기 (U 접두사)
{
    GENERATED_BODY()
};

class NBC_CH3_5_API IItemInterface          // ← 실제 함수가 들어가는 곳 (I 접두사)
{
    GENERATED_BODY()
public:
    virtual void OnItemStartOverlap(...) = 0;   // 순수 가상 함수
    virtual void OnItemEndOverlap(...)   = 0;
    virtual void ActivateItem(AActor* Activator) = 0;
    virtual FName GetItemType() const    = 0;
};
```

**배운 점**
- `U...`(UInterface) + `I...`(실제 인터페이스) 한 쌍으로 선언한다. 언리얼 리플렉션 시스템 때문.
- `= 0` (순수 가상)으로 두면 "이 인터페이스를 구현하는 클래스는 반드시 정의해야 한다"는 계약이 된다.
- 구현 클래스는 `public IItemInterface`로 다중 상속한다 (`ABaseItem : public AActor, public IItemInterface`).

## 1-2. 아이템 베이스 (`ABaseItem`)

모든 아이템의 공통 뼈대. **컴포넌트 구성 + 오버랩 이벤트 바인딩 + 공통 동작**을 담당.

```cpp
ABaseItem::ABaseItem()
{
    Scene = CreateDefaultSubobject<USceneComponent>(TEXT("Scene"));
    SetRootComponent(Scene);

    Collision = CreateDefaultSubobject<USphereComponent>(TEXT("Collision"));
    Collision->SetCollisionProfileName(TEXT("OverlapAllDynamic"));
    Collision->SetupAttachment(Scene);

    StaticMesh = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("StaticMesh"));
    StaticMesh->SetupAttachment(Collision);

    // 오버랩 델리게이트 바인딩
    Collision->OnComponentBeginOverlap.AddDynamic(this, &ABaseItem::OnItemStartOverlap);
    Collision->OnComponentEndOverlap.AddDynamic(this, &ABaseItem::OnItemEndOverlap);
}

void ABaseItem::OnItemStartOverlap(...)
{
    if (OtherActor && OtherActor->ActorHasTag("Player"))
        ActivateItem(OtherActor);     // 플레이어가 닿으면 발동 (다형성!)
}

void ABaseItem::ActivateItem(AActor* Activator) { }   // 기본 빈 구현
FName ABaseItem::GetItemType() const { return ItemType; }
void  ABaseItem::DestroyItem() { Destroy(); }
```

**배운 점 / 설계 포인트**
- **컴포넌트 계층**: `Scene(root) → Collision(Sphere) → StaticMesh`. 루트를 빈 SceneComponent로 두는 패턴.
- **콜리전 프로파일** `OverlapAllDynamic` → 물리 충돌 없이 겹침(Overlap) 이벤트만 발생.
- 오버랩 콜백을 `AddDynamic`으로 바인딩. (대상 함수는 `UFUNCTION`이어야 동작)
- `ActorHasTag("Player")`로 플레이어만 필터링 → **플레이어에 "Player" 태그 세팅이 필요**.
- `OnItemStartOverlap`에서 `ActivateItem()`을 호출 → 자식이 오버라이드한 동작이 다형적으로 실행됨.

## 1-3. 코인 계층 (`CoinItem` → `SmallCoinItem` / `BigCoinItem`)

```cpp
// CoinItem : 코인 공통 베이스 (점수 값 보유)
ACoinItem::ACoinItem() { PointValue = 0; ItemType = "DefaultCoin"; }

void ACoinItem::ActivateItem(AActor* Activator)
{
    if (Activator && Activator->ActorHasTag("Player"))
        GEngine->AddOnScreenDebugMessage(-1, 2.f, FColor::Yellow,
            FString::Printf(TEXT("Player Gained %d Points"), PointValue));
    DestroyItem();
}

// SmallCoinItem : 값만 다르게
ASmallCoinItem::ASmallCoinItem() { PointValue = 10; ItemType = "SmallCoin"; }
// BigCoinItem
ABigCoinItem::ABigCoinItem()   { PointValue = 50; ItemType = "BigCoin"; }
```

**배운 점**
- 중간 베이스(`CoinItem`)에 공통 로직, 자식은 **데이터(PointValue/ItemType)만** 다르게 → 상속 계층 활용.
- `AddOnScreenDebugMessage`로 화면에 디버그 메시지 출력. (점수 시스템 붙기 전 임시 확인용)

## 1-5. 지뢰 아이템 (`MineItem`) — 타이머 + 범위 데미지

가장 복잡한 아이템. **즉시 발동이 아니라 딜레이 후 폭발**한다.

```cpp
AMineItem::AMineItem()
{
    ExplosionDelay = 5.0f; ExplosionRadius = 300.0f; ExplosionDamage = 30.0f;
    ItemType = "Mine";

    ExplosionCollision = CreateDefaultSubobject<USphereComponent>(TEXT("ExplosionCollision"));
    ExplosionCollision->InitSphereRadius(ExplosionRadius);     // 폭발 범위
    ExplosionCollision->SetCollisionProfileName(TEXT("OverlapAllDynamic"));
    ExplosionCollision->SetupAttachment(Scene);
}

void AMineItem::ActivateItem(AActor* Activator)
{
    // 밟으면 즉시 터지지 않고 타이머 예약
    GetWorld()->GetTimerManager().SetTimer(
        ExplosionTimerHandle, this, &AMineItem::Explode, ExplosionDelay, false);
}

void AMineItem::Explode()
{
    TArray<AActor*> OverlappingActors;
    ExplosionCollision->GetOverlappingActors(OverlappingActors);  // 범위 내 액터 수집
    for (AActor* Actor : OverlappingActors)
        if (Actor && Actor->ActorHasTag("Player"))
            GEngine->AddOnScreenDebugMessage(..., FColor::Red,
                FString::Printf(TEXT("Player Gained %f Damage"), ExplosionDamage));
    DestroyItem();
}
```

**배운 점**
- `FTimerManager::SetTimer`로 N초 뒤 함수 호출 예약. (`FTimerHandle`로 핸들 보관)
- 폭발 범위 판정용 **별도 SphereComponent**(`ExplosionCollision`)를 둠. `InitSphereRadius`로 반경 설정.
- `GetOverlappingActors`로 범위 안 액터 일괄 조회 → 범위 데미지 패턴.
- ⚠️ 주의: 생성자에서 `InitSphereRadius(ExplosionRadius)`를 호출하므로, 디테일 패널에서 `ExplosionRadius`를 바꿔도 **콜리전 반경은 생성자 값으로 고정**됨(런타임 반영하려면 별도 처리 필요).

# 파트 2. Pawn 기반 캐릭터 이동 / 중력

## 2-1. 설계 결정

헤더에는 비행(flying) 핸들러(`UpDown`/`Roll`/`AddThrust`), 컨트롤러에는 `IA_Jump`가 있어 **설계가 충돌**했다.
→ "지면 캐릭터 + 점프/중력" 방식으로 통일, 비행용 핸들러 제거.
→ **CharacterMovementComponent를 쓰지 않고** 속도/중력/충돌을 직접 구현.

## 2-2. 이동 파이프라인 (`Tick`)

```cpp
void AMyPawn::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    PerformGroundCheck();                              // 1) 지면 감지

    if (bIsGrounded) { if (Velocity.Z < 0) Velocity.Z = 0; }   // 2) 착지 시 하강속도 제거
    else             { Velocity.Z -= GravityZ * DeltaTime; }   //    공중에선 중력 누적

    // 3) control rotation Yaw 기준 수평 이동
    const FRotator YawRotation(0, GetControlRotation().Yaw, 0);
    const FVector Forward = FRotationMatrix(YawRotation).GetUnitAxis(EAxis::X);
    const FVector Right   = FRotationMatrix(YawRotation).GetUnitAxis(EAxis::Y);
    const FVector H = (Forward * MoveInput.Y + Right * MoveInput.X) * MoveSpeed;
    Velocity.X = H.X; Velocity.Y = H.Y;

    // 4) 스윕 이동 + 벽 슬라이드
    FHitResult Hit;
    AddActorWorldOffset(Velocity * DeltaTime, true, &Hit);
    if (Hit.bBlockingHit)
        Velocity = FVector::VectorPlaneProject(Velocity, Hit.ImpactNormal);

    MoveInput = FVector2D::ZeroVector;                 // 5) 입력 소비
}
```

**핵심**
- **중력 = 속도에 가속도 누적**: `Velocity.Z -= GravityZ * DeltaTime` (`GravityZ = 980`, 언리얼 기본값과 동일).
- 모든 속도/가속도에 **`* DeltaTime`** → 프레임 독립 물리.
- `AddActorWorldOffset(..., bSweep=true)`로 충돌 처리, 충돌 시 `VectorPlaneProject`로 벽 따라 슬라이드.
- 입력은 핸들러에서 채우고 Tick에서 소비 후 0 초기화(consume-and-clear).

## 2-3. 지면 감지 (LineTrace)

```cpp
void AMyPawn::PerformGroundCheck()
{
    const float HalfHeight = CapsuleComp->GetScaledCapsuleHalfHeight();
    const FVector Start = GetActorLocation();
    const FVector End   = Start - FVector(0, 0, HalfHeight + GroundCheckDistance);

    FHitResult Hit;
    FCollisionQueryParams Params;
    Params.AddIgnoredActor(this);     // ← 자기 캡슐 무시 (안 하면 항상 true)

    bIsGrounded = GetWorld()->LineTraceSingleByChannel(Hit, Start, End, ECC_Visibility, Params);
}
```

## 2-4. 점프

```cpp
void AMyPawn::StartJump(const FInputActionValue& Value)
{
    if (bIsGrounded) { Velocity.Z = JumpZVelocity; bIsGrounded = false; }
}
```

# 파트 3. 입력 / 컨트롤러 / 게임모드

## 3-1. Enhanced Input (`MyPlayerController`)
- 컨트롤러가 `InputMappingContext` + `IA_Move`/`IA_Look`/`IA_Jump`를 **소유**.
- `BeginPlay`에서 `UEnhancedInputLocalPlayerSubsystem`에 매핑 컨텍스트 등록.
- 폰은 `SetupPlayerInputComponent`에서 컨트롤러를 캐스팅해 IA를 가져와 바인딩.
  - `IA_Move`/`IA_Look` → `Triggered`, `IA_Jump` → `Started`

```cpp
UEnhancedInputComponent* EIC = Cast<UEnhancedInputComponent>(PlayerInputComponent);
AMyPlayerController* PC = Cast<AMyPlayerController>(GetController());
EIC->BindAction(PC->IA_Move, ETriggerEvent::Triggered, this, &AMyPawn::Move);
EIC->BindAction(PC->IA_Look, ETriggerEvent::Triggered, this, &AMyPawn::Look);
EIC->BindAction(PC->IA_Jump, ETriggerEvent::Started,   this, &AMyPawn::StartJump);
```

## 3-2. 게임모드 (`MyGameMode`)
- 현재는 `AGameMode` 상속 골격만 존재. (DefaultPawn/PlayerController 지정 용도)

# 파트 4. 트러블슈팅 & 핵심 개념 정리

## 4-1. 링커 오류 — 선언만 하고 정의 없는 가상 함수
```
error LNK2001: unresolved external symbol "ACoinItem::ActivateItem"
fatal error LNK1120: 1 unresolved externals
```
- **원인**: `CoinItem.h`에 `virtual ... override`로 선언했지만 `.cpp`에 구현이 없었음.
- `virtual+override` 선언 → vtable 슬롯 생성 → 구현 없으면 **링크 단계**에서 실패(컴파일은 통과).
- **해결**: `CoinItem.cpp`에 `ActivateItem` 구현 추가. (또는 부모 구현을 쓸 거면 선언 삭제)

## 4-2. "카메라는 도는데 폰 본체는 안 돈다"
| 함수/플래그 | 영향 대상 |
|---|---|
| `AddControllerYawInput()` | **Controller**의 control rotation |
| `bUseControllerRotationYaw` (Pawn) | **Pawn(Actor)** 회전을 control rotation에 동기화 |
| `SpringArm->bUsePawnControlRotation` | **카메라 암**을 control rotation에 동기화 |

- **원인**: `AddControllerYawInput`은 control rotation만 바꾸고, 폰의 `bUseControllerRotationYaw`는 `APawn` 기본값 `false`. 카메라는 스프링암 `bUsePawnControlRotation=true` 때문에만 돌고 있었음.
- **해결**: 생성자에 `bUseControllerRotationYaw = true;` (Pitch/Roll은 false → 몸이 위아래로 안 기울게, Pitch는 카메라 암이 담당).

## 4-3. 기타 메모
- 언리얼 인터페이스는 `U...`/`I...` 한 쌍으로 선언.
- 오버랩 콜백 대상 함수는 `UFUNCTION`이어야 `AddDynamic` 바인딩 가능.
- 아이템 발동 대상 필터는 `ActorHasTag("Player")` → 플레이어에 태그 세팅 필요.
